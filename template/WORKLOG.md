# Smart Rest Template Worklog

## Current goal

Build a static HTML template in `template/` that is visually close to `smart-rest-for-ai`.

The workflow requested by the user:

1. Show every sidebar page.
2. Make every sidebar item a separate HTML page.
3. Work page by page.
4. Work only on the page the user names.
5. Move to the next page only after the user says the current page is correct.

## Generated structure

Pages are generated by:

```bash
node template/build-pages.js
```

Do not manually edit generated HTML pages for persistent changes. Edit:

- `template/build-pages.js`
- `template/assets/template.css`
- `template/assets/template.js`

Then rerun the generator.

`template/PAGES.md` is generated and lists all created pages.

## Current implementation status

- Every generated page includes the same full sidebar.
- Horizontal scroll rule for wide pages/tables: match the source behavior shown on report pages. The horizontal scrollbar must belong to the browser window/body and start from the far left of the viewport, under the sidebar as well. Do not create inner horizontal scrollbars on `#main-content`, `.wrapper`, `.grid-view`, `.table-responsive`, DataTables wrappers, or custom table wrappers. For wide tables, let the table overflow naturally and override page-specific shell styles as needed so `body/html` owns the horizontal scroll (`.wrapper`/content containers should use `overflow: visible`, not `overflow: scroll/auto`). `reports-delivery.html` is the current reference implementation for this scroll behavior.
